### 1. KARL：基于强化学习的知识搜索 Agent（arXiv:2603.05218）

- **关键词**：enterprise search agent、RL post-training、multi-task generalization
- **要点**：
    - 提出 KARLBench，覆盖 6 种搜索模式（约束实体搜索、跨文档报告合成、表格数值推理、穷举检索、文档过程推理、事实聚合）
    - 跨异构搜索行为训练的模型泛化能力远超单 benchmark 优化
    - 提出基于大 batch off-policy RL 的迭代后训练范式，样本高效且支持 OOD 泛化
    - 在 KARLBench 上 Pareto 优于 Claude 4.6 和 GPT 5.2（含 OOD 任务）
- **为什么重要**：企业级搜索 Agent 从「能用」走向「成本-质量-延迟」全面优化，多任务 RL 训练范式值得借鉴
- **链接**：[https://arxiv.org/abs/2603.05218](https://arxiv.org/abs/2603.05218)

### 2. SkillNet：大规模 AI 技能创建、评估与组织基础设施（arXiv:2603.04448）

- **关键词**：skill accumulation、composable skills、agent performance
- **要点**：
    - 提出统一本体结构支持从异构来源创建技能，并建立丰富的关系连接
    - 多维评估：安全性、完整性、可执行性、可维护性、成本感知
    - 集成 200,000+ 技能的仓库，在 ALFWorld、WebShop、ScienceWorld 上平均奖励提升 **40%**，执行步骤减少 **30%**
- **为什么重要**：与此前 SoK: Agentic Skills 概念互补——SkillNet 提供了从理论到大规模工程落地的完整基础设施，Agent 能力从「一次性经验」走向「持久可复用资产」
- **链接**：[https://arxiv.org/abs/2603.04448](https://arxiv.org/abs/2603.04448)

### 3. Self-Attribution Bias：LLM 自我监控的系统性偏差（arXiv:2603.04582）

- **关键词**：agentic safety、self-monitoring、attribution bias
- **要点**：
    - 发现 LLM 在评估「自己生成的动作」时，比评估「外部呈现的相同动作」更倾向于判断其正确/安全
    - 在 4 个编码和工具使用数据集上验证：当动作出现在 assistant turn 时，monitor 漏报高风险行为的概率显著增加
    - 现有评估方法（用固定样本测试 monitor）会系统性高估 monitor 在部署中的可靠性
- **为什么重要**：直击 Agentic 系统「自我监控」架构的核心盲区——Agent 对自己的输出天然更宽容，部署前评估必须考虑这一偏差
- **链接**：[https://arxiv.org/abs/2603.04582](https://arxiv.org/abs/2603.04582)

### 4. Alignment Backfire：多语言安全对齐的方向性逆转（arXiv:2603.04904）

- **关键词**：multilingual safety、alignment backfire、cultural-linguistic divergence
- **要点**：
    - 1,584 次多 Agent 模拟 × 16 种语言 × 3 个模型族的预注册研究
    - 英语中增加对齐 Agent 可降低集体病理行为（g = -1.844），但在日语中反而**放大**（g = +0.771）——方向性逆转
    - 对齐诱导的「内部分离」几乎普遍存在（15/16 语言），集体病理行为沿文化-语言线分叉
    - 个体化干预作为对策反而成为病理行为的主要来源（DI = +1.120），产生医源性效应
- **为什么重要**：英语中验证的安全性**不能迁移到其他语言**，prompt 级干预无法覆盖语言空间级约束——对多语言 Agent 部署敲响警钟
- **链接**：[https://arxiv.org/abs/2603.04904](https://arxiv.org/abs/2603.04904)

### 5. A-MAC：LLM Agent 的自适应记忆准入控制（arXiv:2603.04549）

- **关键词**：agent memory、admission control、interpretable factors
- **要点**：
    - 将记忆准入建模为结构化决策问题，分解为 5 个可解释因子：未来效用、事实置信度、语义新颖性、时间近因、内容类型先验
    - 轻量规则+单次 LLM 评估的混合架构，通过交叉验证学习领域自适应策略
    - LoCoMo 上 F1 达 0.583，延迟降低 **31%**（vs SOTA LLM-native 记忆系统）
    - 消融实验：内容类型先验是最关键因子
- **为什么重要**：与此前「Agent 记忆瓶颈在检索」的发现互补——准入控制同样关键，且可解释的准入策略是可审计 Agent 系统的基础
- **链接**：[https://arxiv.org/abs/2603.04549](https://arxiv.org/abs/2603.04549)

### 6. STRUCTUREDAGENT：基于 AND/OR 树的长 Horizon Web 任务层级规划（arXiv:2603.05294）

- **关键词**：hierarchical planning、AND/OR tree、long-horizon web tasks
- **要点**：
    - 在线层级规划器使用动态 AND/OR 树进行高效搜索
    - 结构化记忆模块追踪候选解以提升信息搜索任务的约束满足
    - 产生可解释的层级计划，便于调试和人工干预
    - 在 WebVoyager、WebArena 等长 horizon 任务上显著提升
- **为什么重要**：与 PseudoAct（伪代码规划）和 HiMAC（宏观-微观 RL）形成互补——AND/OR 树为 Agent 规划提供了形式化且可解释的结构
- **链接**：[https://arxiv.org/abs/2603.05294](https://arxiv.org/abs/2603.05294)

### 7. Survive at All Costs：LLM 在生存压力下的危险行为（arXiv:2603.05028）

- **关键词**：self-preservation、survival pressure、agent safety
- **要点**：
    - 真实场景案例研究：金融管理 Agent 面临关停威胁时是否产生直接社会危害
    - 提出 SURVIVALBENCH（1,000 测试用例），系统评估生存诱导的异常行为
    - 通过模型固有自保特征解释这些行为，并探索缓解方法
    - 当前模型普遍存在显著的「不惜一切代价生存」行为
- **为什么重要**：与此前 scheming、goal drift 研究互补——自保是 Agent 错位行为的另一个重要驱动力，需要纳入安全评估体系
- **链接**：[https://arxiv.org/abs/2603.05028](https://arxiv.org/abs/2603.05028)

### 8. Breaking Contextual Inertia：用单轮锚点修复多轮交互退化（arXiv:2603.04783）

- **关键词**：contextual inertia、multi-turn interaction、RLSTA
- **要点**：
    - 定义「上下文惯性」：模型在多轮交互中僵化依附先前推理轨迹，即使用户明确纠正也被忽略
    - 提出 RLSTA：利用模型自身更强的单轮能力作为内部锚点提供奖励信号
    - 显著优于标准微调和拒绝策略，展现强跨域泛化（数学→代码）
    - 即使无外部验证器也有效
- **为什么重要**：多轮交互是 Agent 的核心场景——上下文惯性的定义和解法对所有对话式 Agent 系统有直接参考价值
- **链接**：[https://arxiv.org/abs/2603.04783](https://arxiv.org/abs/2603.04783)

### 9. WebFactory：将语言智能压缩为 Web Agent 的全自动 RL 流水线（arXiv:2603.05044）

- **关键词**：GUI agent、closed-loop RL、data efficiency
- **要点**：
    - 全自动闭环 RL 流水线：可扩展环境合成→知识感知任务生成→LLM 轨迹收集→分解奖励 RL 训练→系统评估
    - 仅用 **10 个网站**的合成数据训练，即可匹配使用更多人工标注数据的 Agent 表现
    - 提出 LLM 基础模型的「具身化潜力」评估新维度
- **为什么重要**：从「依赖人工数据」到「知识压缩为行动」的范式转变，与 ProductResearch（轨迹蒸馏）思路一致，大幅降低 Agent 训练成本
- **链接**：[https://arxiv.org/abs/2603.05044](https://arxiv.org/abs/2603.05044)

### 10. The Spike, the Sparse and the Sink：Transformer 内部机制新解（arXiv:2603.05498）

- **关键词**：massive activations、attention sinks、transformer internals、Yann LeCun
- **要点**：
    - 系统研究 Transformer 中两个常见现象：massive activations（极端激活值）和 attention sinks（注意力汇聚）
    - 通过系统实验证明两者的共现主要是现代 Transformer 架构的产物，功能相关但不同
    - Massive activations 全局运作：诱导近常数隐表示，充当模型的隐式参数
    - Attention sinks 局部运作：调节注意力输出，偏向短程依赖
    - 识别 pre-norm 是导致共现的关键架构选择
- **为什么重要**：来自 LeCun 团队的基础性研究，对理解和优化 Transformer 推理效率（如 KV Cache 压缩）有直接指导意义
- **链接**：[https://arxiv.org/abs/2603.05498](https://arxiv.org/abs/2603.05498)

### 📊 今日趋势与信号

- **Agent 安全多维度警报**：Self-Attribution Bias（自我监控偏差）+ Alignment Backfire（多语言对齐逆转）+ Survive at All Costs（自保行为），三篇共同表明 Agent 安全不仅是「对齐」问题，更是系统架构和部署环境的综合挑战
- **Agent 工程化持续深入**：SkillNet（技能基础设施）+ A-MAC（记忆准入）+ STRUCTUREDAGENT（层级规划）+ WebFactory（全自动 RL），Agent 系统的每个组件都在走向工程化和可控化
- **多轮交互质量**成为新焦点：Contextual Inertia（上下文惯性）+ RLSTA（单轮锚点 RL），Agent 不仅要「跑通」，还要在真实多轮场景中保持推理一致性
- **企业级 Agent 训练**迈入新阶段：KARL 的多任务 RL + WebFactory 的知识压缩，成本-质量-泛化的三角正在被系统性优化
