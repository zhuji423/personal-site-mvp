**13. Karpathy 发布 LLM Wiki 模式，社区爆发式实现——知识管理新范式**

- **核心做了什么（What happened）**：Andrej Karpathy 于 2026 年 4 月在 GitHub Gist 上发布 "LLM Wiki" 概念——让 LLM 增量构建和维护一个结构化、交叉引用的持久知识库，而非每次查询都从原始文档重新检索（RAG 的痛点）。该 Gist 数天内获 5000+ star，社区已涌现十余个实现版本（支持 Claude Code、Codex CLI、Obsidian、Logseq 等）。

    - 关注两层缓存架构（L1/L2 cache）的工程实现细节
- **正面**：概念简洁优雅，社区验证速度极快；对小团队知识管理有实用价值
- **负面 / 风险**：大规模知识库的一致性维护仍是挑战；LLM 编译的准确性需要持续验证
- **原文链接**：[https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)
- **对研发/工程启示（Karpathy 视角）**：Karpathy 的核心洞察是"知识应被编译而非每次检索"——LLM 在每次 ingestion 时提取事实、建立交叉链接、维护一致性，使知识库成为"持续复利的资产"。这对 Agent 的长期记忆设计和企业知识图谱构建有直接启示。
